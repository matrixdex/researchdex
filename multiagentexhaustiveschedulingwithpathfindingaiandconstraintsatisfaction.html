Abstract

Filmmaking involves scheduling of shots across the entire shooting period, taking into account factors like actor schedules, and location permissions. We attempt to recognize the combinatorial explosion problem at the heart of this and interpret it as a pathfinding problem. Strategies like constraint satisfaction, greedy, depth-first search, breadth-first search and A* search are used to efficiently solve this problem. The final algorithm is used to create an AI that can schedule the shoot of an entire film in 30 seconds, as opposed to the incumbent practice of doing it manually, which takes at least a week. In practice, this allows filmmakers to focus on their art, while the mundane aspects are automated. This algorithm is also extensible to other scheduling tasks.

Introduction

A. Current state of shoot scheduling

A 1-hour feature film may consist of 250 individual shots, spread over multiple locations. Currently, the scheduling of these shots across entire shooting period (say 2 months) is done manually, noting actor availability, location availability, equipment availability, and so on. This laborious task usually takes more than a week. We recognize the combinatorial problem at the core and try various approaches to solve it efficiently.

B. Problem formulation

B.(i) Combinatorial explosion

A shoot schedule is simply an arrangement of shots. This arrangement must satisfy all constraints and minimize usage of budget. Trying to find most efficient shoot schedule is thus a combinatorial explosion problem [1].

B.(ii) Interpreting problem as exhaustive pathfinding

A path that traverses all shots with minimum budget (cost) needs to be found. Constraint satisfaction can be done independent of this path, fixing certain shots at certain positions, after which a path can be found. Each shot is interpreted as a location in pathfinding problem.

B.(iii) Interpreting shot as location

Each shot consists of actors and equipment. For simplicity, only actors are taken into account (equipment follows same principle). A shot is interpreted as a set of actors, each actor being an independent element.

B.(iv) Interpreting distance in pathfinding problem

Distance governs the heuristic and is necessary to find efficient path. We interpret distance between shots using Hamming distance [2]. Since each shot is a set of actors, the distance is a function of the number of actors in one set but not the other, and vice versa (difference between sets).

B.(v) Assumptions

1. Actors are independent of each other.

2. Shoot is done sequentially, no shot is done in parallel.

3. Each actor has a fixed price per day.

4. Each location has a fixed price per day.

5. A calendar day can have 3 shots taken in it.

6. Actors are the source of most expense, hence optimizing schedule for actors’ time will lead to most efficient budgets. This was tested in dummy schedules with less than 20 actors; actors with high per-day costs were grouped together.

B.(vi) Goal

We wish to build an algorithm that generates an optimum shoot schedule in polynomial time.

Results

A. Functions and variables used

• totalExpense(shootSchedule): Returns total actor expense

of entire shoot schedule by multiplying actor cost per day

and number of days each actor works

• minimumExpense(film): Returns minimum actor expense

of film by multiplying actor cost per shot and number of

shots each actor is in. Actor cost per shot = actor cost

per day / 3

• currentExpense(shootSchedule): Returns expense of all

actors till now of given shoot schedule, if remaining shots

are not shot.

• totalShots = total number of shots in film

• scheduledShots = number of shots already scheduled

• percentShotsDone = scheduledShots/totalShots

B. Simple constraint satisfaction of actors and locations

Because of the constrained nature of an optimal schedule,

we first chose to interpret this problem as a constraint satisfac-

tion problem [3]. Each node was a shot. A particular day and

time (out of 3 per day) could be assigned to each node/shot,

with constraints being actors: the same actor could not be in

the two shots at the same time.

Actor availability and location availability were given as

input and a shoot schedule was generated, with each shot

satisfying the actor and location constraints of its set of actors

and the location it needed to be shot at. A shoot schedule was

generated by assigning nodes a number (a function of day and

time). Backtracking was also used.

This was inefficient because many shoot schedules could

be generated that fulfilled all these constraints. This was

because in most films, actors are hired for a long duration of

time, hence they can easily fill schedules. The same goes for

location. This method was little better than randomly guessing

a shoot schedule.

C. Greedy pathfinding

This approach used greedy pathfinding [4]. The node with

shortest Hamming distance was chosen. In case of multiple

nodes with same distance, a random node among them is

chosen.

Initial node is chosen randomly. This algorithm generates

a path following minimal changes to casting in sequences of

shots. It is exhaustive since all shots need to be scheduled, i.e.

all locations must be visited.

It does not give optimal solution, being greedy in nature.

But it does lead to certain insights like clustering and heuristics

which are used to further refine algorithm.

Another problem with this algorithm is that it wastes

computing power by random selection of shots with same

actors, i.e. 0 Hamming distance.

Finally, locations are not taken into account in this approach.

D. Clustering

We noticed in dry runs while minimizing for budget using

exhaustive search that shots with same actors were usually

scheduled together. This is because clustering actors together

allows their total number of hired days to decrease. Since

clustering in a polynomial time operation, we use it before

running the search algorithm for optimization.

2 kinds of clustering are used:

1) Location clustering: Shots of one location are all

grouped together. Each location is a set containing the

actors needed at that location. A path is generated

between locations using the same Hamming distance.

2) Shot clustering: Within each location, shots with 0

Hamming distance, i.e. containing the same set of actors,

are grouped together. This set is used in generating shoot

schedule with pathfinding methods. Each node in this set

has a different set of actors.

Using the above clustering, a method is determined. The

shots with more expensive actors are clustered together first,

minimizing the number of days they are hired for. This is

a hardcoded method that generates shoot schedules that are

cheaper than the average budget across all possible shoot

schedules.

E. Clustering with A* search

The hardcoded method used above still does not provide an

optimal shoot schedule. Thus we use A* search [5] on the set

that contains nodes with different sets of actors.

Heuristic (h): currentEx-

pense(shootSchedule)/percentShotsDone This heuristic is

minimized. It represents the cost of complete shoot schedule

assuming it rises linearly as percentage of scheduled shoot

increases. These parameters together make algorithm choose

the node that generates minimum expense for maximum

number of shots.

Path travelled (g): currentExpense(shootSchedule) This is

the current expense of film, i.e. if film was to only shoot the

already scheduled shots.

Using these 2 functions in A* search provides a roughly op-

timal solution. The comparison between nodes (shots) occurs

by comparing a rough estimate of the total shoot expense,

while current expense maintains a record of how efficient

the current path is, which was not possible in depth-first and

breadth-first approaches. This algorithm is used exhaustively

with end condition being the initial set becoming empty, i.e.

all nodes have been scheduled. A unique number is given to

each node, which is the sequence number. This is used to order

the shoot in calendar dates, taking into account holidays and

other nuances.

F . BFS, DFS and other notes

Greedy breadth-first [6] and depth-first searches [7] were

also implemented with simply the heuristic. These provided

suboptimal results as they could not keep track of history of

already scheduled shots.

Constraints on location such as availability can be adjusted

for by fixing certain shoots on certain days, i.e giving them

a particular sequence number. These are initial conditions

around which clustering is done, and further A* search deter-

mines schedule with these sequence numbers fixed. Sequence

numbers are simply a function of the calendar, not shoot

schedule, hence can be predetermined.

IV. R ESULTS

The results for different methods are given below.

A. Simple constraint satisfaction

• Only takes into account actor and location availability,

which leads to a broad set of possible solutions

• Many possible solutions result in mostly random selection

• Thus inefficient when optimizing for budget

• Optimal solution not generated but simple framework

works well to produce quick results

B. Greedy pathfinding

• Greedily chooses nodes based on minimum Hamming

distance

• Algorithm wastes time on nodes with zero Hamming

distance

• Greedy algorithm does not lead to optimal solution

• Simple framework works well to produce quick results

C. Simple clustering and hardcoded algorithm

• Clusters simplify initial set/graph, very important factor

• Minimizing expensive actor costs sometimes work well

in best case where most expensive actor is a lot more

( 100x) expensive than others, but do not produce results

in realistic edge cases

• Does not lead to optimal solution

• Uses insight generated from exhaustive search runcases

D. BFS and DFS

• Using heuristic significantly generalizes algorithm for all

cases including edge cases

• Heuristic alone does not provide algorithm with any in-

formation about effectiveness of shoot schedule generated

at every point, this results in choosing schedules that are

not optimal

• Does not lead to optimal solution

E. A* search

• Heuristic and history combined give an accurate picture

of how efficient the current shoot schedule is

• Heuristic is admissible as it almost always underestimates

cost. This is because it assumes linearity in estimating

total expense, but due to the addition of extra actors (since

all members of the set are different), the cost will increase

more than linearly.

• Leads to roughly optimal solution

V. C ONCLUSION

A* search generates a solution that is roughly optimal. It

is a good approximation because of the inadmissibility of

heuristic function in certain edge cases. A new approach using

opportunity cost is underlined and can lead to a more efficient

algorithm.

VI. D ISCUSSION

The given problem attempts to tackle combinatorial explo-

sion using a variety of optimization and pathfinding methods.

A limitation of these methods that we have come across

is that the assumption of an admissible heuristic. Towards

end nodes, when all actors have been accounted for and

no new actors come into shots, it overestimates the total

expense. Furthermore, this problem is very similar to travelling

salesman [8] and requires a new approach.

A. Further research

Opportunity cost is the cost of choosing a particular node to

traverse to. This is interpreted in fully connected graphs as a

way to encode the information that any node can be travelled

to from any node. This cost of choosing a node keeps an

account all other nodes that were left out, as well as the fact

that every node ultimately has to be traversed because this is

exhaustive search. This is the next step for this research. We

believe it will also hold insight into the travelling salesman

problem.

VII. F UTURE ENDEAVORS

We’d like to use this algorithm with an OCR program like

Vision API [9] to directly read scripts, generate a list of shots

and actors, and finally output an optimal shoot schedule on a

calendar. This automates the entire process and the filmmaker

only needs to work on their script.
